对文件进行读写操作

  • 打开文件

    • 函数open() 打开一个文件
    • 括号中填写，打开文件的相对路径和打开模式
      • 格式举例：open( file=’test.txt’,mode=’w’) 打开或创建python脚本所在目录中的test.txt文件，并进行编辑。
    • 打开模式可以不写，默认就为read模式
    • 打开模式的分类
      • mode=’r’ 代表打开只读模式，例如：open( file=’test.txt’,mode=’r’)
        • 如果打开的文件不存在，会提示报错
        • 打开文件后，不能使用write相关的函数。
      • mode=’w’ 代表打开写入模式，例如：open( file=’test.txt’,mode=’w’)
        • 如果打开的文件不存在，会新建这个文件
        • 如果打开的文件已存在，写入数据时会覆盖文件中原有的数据。
        • 在该模式下，不能使用read相关的函数
      • mode=’a’ 代表打开追加写入模式，例如：open( file=’test.txt’,mode=’a’)
        • 如果打开的文件不存在，会新建这个文件
        • 如果打开的文件已存在，写入数据会追加到原来的数据后面
        • 在该模式下，不能使用read相关的函数
          • 打开模式只能选择一种，python中对于一个文件操作不能读他的同时还要求编写他。
          • 当打开模式为r时，只能使用读文件的相关函数，写文件无法使用
          • 当打开模式为w或a时，只能使用写文件的相关函数，读文件不能使用

  • 关闭文件

    • 函数close() 保存并关闭文件

  • 写文件

    • 函数write() 向文件里写数据，括号中可以写数字或字符串或其他。
      • 例如: write(“i love china\n”) 代表写入信息i love china,并且换行

  • 读文件

    • 函数readline() 读文本中的一行内容
    • 函数read() 读文本中的数据
      • 括号中的参数可以填写整数，表明读多少行，例如:read(50) 代表读50行数据
      • 括号中的参数不填写的话，代表读所有行，如果之前使用过readline读取过，已经读取过的不会被再次读取一遍

文件操作中其他必要操作

  • seek() 字节移动，在括号中填写数字，代表从第几个字节后的内容开始进行操作。
    • 比如 seek(2)，代表选取从第二个字节后的内容
  • seekable() 查看打开的文件能否使用seek命令
  • tell() 反馈seek字节移动的位置
  • flush() 将缓存中的数据强制刷到硬盘上
    • 当向一个文件写入数据时，如果不进行close()操作，写入的数据都是存在缓存里的，不会写入到文件中
    • 除非等缓存空间满了，才会自动刷新到文件中，也就是落盘
    • 如果正在写入重要的数据到文件中，但是迟迟不会进行close()操作，就可以使用flush()函数，就可以周期性的将数据刷到硬盘中。
  • truncate() 截取文件内容，括号中填写数字用于指定长度
    • truncate(100) 截取文件第一个字节到第100个字节，100以后信息的全都过滤掉
    • truncate(） 不填写参数就代表全部从光标所在字节之后的信息全都过滤掉

文件操作的混合模式

  • w+ 代表创建一个文件，并可以执行写入操作，也可以执行读操作
  • r+ 代表读写，打开这个文件后，读文件时，会从头读，读的是原来的内容，写的时候是追加写
  • a+ 代表追加读，打开这个文件后，读文件时，读光标会移动到内容最后，读不出任何数据，写的时候是追加写。
  • 混合模式很少使用，一般都单独写或单独读。

名称空间

  • 定义：程序中的对象是放到内存里的，而变量与对象的对应关系是存在名称空间里的，也可以说变量都放在名称空间中
  • 种类：有四种
    • locals：函数内部的名字空间，存储函数内的局部变量和形参，常用
    • globals：整个模块的名字空间，模块就是python文件，存储全局变量，常用
      • 可以在函数内部中声明一个变量为全局变量，但是不建议使用 使用格式：global name
    • enclosing：在嵌套函数中外部函数的名字空间，如func1函数里用了func2函数，那么对于func2函数而言，fun1的名字空间就是enclosing
    • builtins：内置模块空间，里面存储提前预置号的变量或函数
      • 变量所在的名称空间决定了它的作用域，也就是在哪些范围内可以生效。
      • 一般变量在globals时，就是全局存活，全局有效
      • 一般变量在locals，就是局部存储，局部有效
      • 查看作用域的方法
      • globals()函数，可以查看到一个function函数内的所有变量
      • locals()函数，可以查看到整个python中所有的变量
      • 作用域查找的顺序
      • 当程序使用某个变量的时候，需要查找这个变量在那个名字空间中，从而知道它的作用范围是多大
      • 搜索顺序为LEGB
        • 先搜索Locals
        • 再搜索enclosing
        • 之后搜索globals
        • 最后搜索builtins
        • 如果都没有找到这个变量名称，就会调用内置空间builtins里的NameError，给执行者抛出NameError异常提示
  • 了解名称空间的目的
    • 当在程序的某一处需要使用一个变量，了解名称空间及其查找顺序的话，即使这个变量在之前的多个地方都与不同的对象进行绑定，那么也知道此处需要引用的变量实际的对象是什么，
    • 也可以排查出当出现无法找到变量时，具体应该在哪个名称空间中去创建才合理。

闭包现象

  • 定义：当函数func1中嵌套了func2，并且func2中引用了func1的变量，如果后续程序会使用func2，不会再使用func1，那么本应该销毁掉func1里面的所有变量，由于这中现象的方式就不会被销毁，其名字空间也不会被释放，这种现象叫做闭包现象。（自己理解的，不是官方定义）
  • 用途： 此概念一般使用在装饰器中

    装饰器

  • 当我们对原有的程序需要新增一个功能时，需要遵循开放-封闭原则
  • 开放封闭原则
    • 封闭：不能为了增加一个新功能，修改原来已经写好并且可运行的代码，也不能修改原来的调用方式
    • 开放：可对原有功能进行拓展，从而实现新增的功能
  • 以实例来讲述装饰器的实现、开放-封闭原则、以及闭包现象的使用。

  • 生成器主要适用于那些，数值给定的范围很大，但是由于判断条件，只会用到一小部分的情况
  • 简单的说，与其将数值可能等于的整个范围加载到内存中，再进行遍历，不如直接通过生成器不停模拟出下一次出现的数字，这样只既节省空间也节省时间。

Python常用模块学习笔记

模块的介绍

  • 什么是模块
    • 将多个函数分组，将相关的函数放到py文件中，正在书写的代码需要函数时直接调用py文件，避免程序代码出现成百上千个函数，体积臃肿
    • 在python中，py文件就可以称之为模块
    • 一个模块可由一个py文件或多个py文件构成
  • 模块的好处
    • 提高代码的可维护性
    • 无论是自己写的模块、第三方的模块、内置的模块都可以被引用到我们的编程中，不必重新写一些基础的代码。
    • 使用模块可以避免函数名和变量名冲突，py中写的函数名和你本体代码中的函数名哪怕重名也是不受影响的，如果都将函数直接写在本体中，就需要考虑
  • 模块的分类
    • 内置模块（标准模块、标准库、近300个）
    • 第三方模块 其他人上传到社区的模块，需要单点安装 pip install
    • 自定义模块 自己写的模块
  • 模块的导入&调用
    • import module_name 导入模块
    • from module import xx 从模块中导入函数，可直接使用函数即可，不需要加模块前缀
    • from module.xx import xx 从模块中某个子模块或分类中导入某个函数
    • from module.xx import xx as xxx 从模块中某个子模块或分类中导入某个函数，并将函数起个别名
    • from os import * 从模块中导入所有函数
  • 模块导入的写入格式
    • 通常将模块导入语句写在开头
    • 语句要顶头写

序列化与反序列化

  • 含义：当我们需要将字典或列表的数据从内存中存入硬盘中时，需要将字典或列表等数据进行序列化后，方可以写入硬盘中，否则写入报错，因为写入硬盘时只能是字符串，不能是其他格式，因此其他格式要写入时必须格式化，当从硬盘中读取数据时，需要对数据进行反序列化，这样就可以还原数据格式
  • pickle模块可以用序列化与反序列化
  • 常用函数
    • pickle.dumps() 为将数据序列化的函数，括号中填写的是需要进行序列化的内容
    • pickle.loads() 为将数据反序列化的函数，括号中填写的是需要进行反序列化的内容
    • pickle.dump(x,f) 为将数据进行序列化并直接存入文件中,x为需要写入的数据，f为写入的文件
    • pickle.load(f) 为将存有序列化的文件进行反序列化解析后进行读取
  • 特征
    • 先进先出：用反序列化进行数据解析时，优先解析的是第一个写入的序列化数据。

hashlib加密模块

  • hash定义：也称为散列，又被音译为哈希，就是把任意长度的输入（又叫做预映射,pre-image），通过散列算法，变成固定长度的输出，该输出就是散列值。这种转换就是一个压缩映射，也就是，散列值的空间通过远远小于输入的空间，不同的输入可能会散列成相同的输出，而不可能通过散列值来唯一的确定输入值的内容。简单的说就是一种将任意长度的消息压缩到某一固定长度的信息摘要的函数。
  • hash用途：主要用于信息安全领域中的加密算法，他把不同长度的信息转换为128位的编码里，叫作hash值，也可以说，hash就是找到一种数据内容和数据存放地址之间的映射关系
  • 但是由于每次使用hash()函数对某个对象加密后，结果都是不同的，因此在hash的基础上衍生了真正的加密算法。
  • 加密算法：
    • MD5
      • 定义：MD5讯息摘要演算法，一种被广泛使用的密码杂凑函数，可以产生出一个128位的散列值，用于确保信息传输的完整一致。MD5的前身有MD2、MD3、MD4
      • 功能：
        • 1、输入任意长度的信息，经过处理，输出128位的信息（数字指纹）
        • 2、不同的输入得到的不同结果（唯一性）
      • 算法的特点：
        • 1、压缩性：任意长度的数据，算出的MD5值的长度是固定的。
        • 2、容易计算：从原数据计算出MD5值很容易
        • 3、抗修改性：对原数据进行改动后，修改字节生成的MD5值区别会很大
        • 4、强抗碰撞：出现hash碰撞的概率极低。
        • 5、不可逆：md5使用的是hash算法，计算过程中原文的部分信息是丢失的
          • 用途:
          • 1、防止被篡改
          • 2、防止直接看到明文
          • 3、防止抵赖（数字签名）
          • 用法：
          • hashlib.md5() 创建一个md5加密队列
          • m.update(“xxx”) 将内容xxx更新到md5加密队列中，等待加密，多次update不同的内容时，内容不是覆盖而是追加到最后。m为创建的加密队列，xxx为需要加密的内容
          • m.digest() 将m中的所有内容进行加密，结果以byte形式呈现
          • m.hexdigest() 将m中的所有内容进行加密，结果以十六进制呈现

  • SHA-1加密算法
    • 定义:
      安全哈希算法，主要使用于数字签名标准里面定义的数字签名算法。对于长度小于2^64位的消息，SHA1会产生一个160位的消息摘要。当接收到消息的时候，这个消息摘要可以用来验证数据的完整性。
    • 历史：
      由于山东大学王小云破解了MD5和sha-1，科学家又推出了SHA224,SHA256,SHA384,SHA512,位数越长越难被破解，一般流行的是sha-256，https的加密算法就是基于sha-256的
    • 用法：
      • hashlib.sha1() 创建一个sha1加密队列
      • hashlib.sha256() 创建一个sha256加密队列
      • m.update(“xxx”) 将内容xxx更新到sha1加密队列中，等待加密，多次update不同的内容时，内容不是覆盖而是追加到最后。m为创建的加密队列，xxx为需要加密的内容
      • m.digest() 将m中的所有内容进行加密，结果以byte形式呈现
      • m.hexdigest() 将m中的所有内容进行加密，结果以十六进制呈现

MD5与SHA-1

  • 由于MD5与SHA-1均是从MD4演化而来。他们的结构和特性有很多的相似之处，SHA-1与MD5最大的区别在于SHA-1的结果要比MD5长32字节。对于强行攻击而言，越长的加密破解的难度就越大。因此SHA–1对抗强行攻击的能力要比MD5强。以此也可以知道SHA-256也要比SHA-1更加牛逼。
  • 当前主流使用MD5对文件的一致性进行校验，确保文件的一致性。由于md5加密速度很快
  • 对于加密而言，一般还是使用sha加密算法

软件开发目录设计规范

  • 推荐的最方便的目录结构，如果项目叫做foo
    • bin/:存放项目的一些可执行文件，当然起名script一类的也行
    • foo/：存放项目的所有源代码
      • 源代码中的所有模块，包都应该在这个目录中
      • 其子目录tests存放单元测试代码
      • 程序的入口最好命名为main.py
    • docs/：存放一些文档
    • conf/:存放配置文件
    • setup.py：安装、部署、打包娇而不能
      • 业界标准的写法是用python流行的打包工具setuptools
      • 但不是说必须要用打包工具，而是建议一个项目一定要有自己的安装部署工具
      • 目的是为了能够快速搭建
    • requirements.txt:存放软件依赖的外部python包列表
      • 方便开发者维护软件的包依赖，将开发过程中新增的包添加进这个列表中，避免setip.py安装依赖时漏掉软件包
      • 方便读者明确项目使用了哪些python包
      • 文件的内容格式为每一行包含一个包依赖的说明，通常是flask>=0.10这种格式
      • 要求这个格式能够被pip识别，这样就可以简单使用pip install -r requirements.txt 命令把所有依赖都安装好。
      • 使用pip freeze > requirements 可以快速将已安装的python包生成requirements文件。
    • README:项目说明文件
      • 软件定位，软件的基本功能
      • 运行代码的方式：安装环境，启动命令
      • 简要的使用功能

字符串的格式化

  • 把字符串按照我们想要的格式进行打印或填充
  • 格式化的方式分类
    • 使用%
      • 类似正则
      • %s 代表字符串
      • %r 字符串，表示正则，不是普通的字符串
      • %c 代表整数转换为单个字符
      • %d 代表数字（整数），如果输入是小数，只取整数
      • %u 代表无符号整数
      • %o 代表八进制
      • %x 代表十六进制 （小写x的时候，十六进制内容中字母要为小写）
      • %X 代表十六进制 （大写X，六进制内容中字母要为大写）
      • %e 代表浮点数（科学计数法，输入值必须要有e，且e为小写）例如3.2e+4
      • %E 代表浮点数（科学计数法，输入值必须要有E，且E为大写）例如3.2E+4
      • %f 代表浮点数（小数）,%与f中添加.1，代表小数点后保留一位数，%.2f 代表小数点后保留两位数
      • %g/%G 代表十进制形式的浮点或指数浮点的自动转换
      • 当格式化的数据与%后的输入值个数不一致时，会出现报错。因此格式化数据与输入值要一一对应。
    • 使用format函数

  • 字符串占位方法
    • %s或%d等格式化数据的输入值需要占位的话，可以在%与字母中间加上数字，例如：%5d 表明占5位，默认右对齐
    • 想要左对齐的话，就加上-。即%5d改为%-5d
    • 想要补0的话，将%5d改为%05d即可。
    • 对于浮点数需要占位的话，使用%f，并在%与f中添加.1，代表小数点后保留一位数，%.2f 代表小数点后保留两位数，从而满足占位的要求
