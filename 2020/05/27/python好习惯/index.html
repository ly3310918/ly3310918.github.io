python好习惯

软件开发目录设计规范

  • 推荐的最方便的目录结构，如果项目叫做foo
    • bin/:存放项目的一些可执行文件，当然起名script一类的也行
    • foo/：存放项目的所有源代码
      • 源代码中的所有模块，包都应该在这个目录中
      • 其子目录tests存放单元测试代码
      • 程序的入口最好命名为main.py
    • docs/：存放一些文档
    • conf/:存放配置文件
    • setup.py：安装、部署、打包娇而不能
      • 业界标准的写法是用python流行的打包工具setuptools
      • 但不是说必须要用打包工具，而是建议一个项目一定要有自己的安装部署工具
      • 目的是为了能够快速搭建
    • requirements.txt:存放软件依赖的外部python包列表
      • 方便开发者维护软件的包依赖，将开发过程中新增的包添加进这个列表中，避免setip.py安装依赖时漏掉软件包
      • 方便读者明确项目使用了哪些python包
      • 文件的内容格式为每一行包含一个包依赖的说明，通常是flask>=0.10这种格式
      • 要求这个格式能够被pip识别，这样就可以简单使用pip install -r requirements.txt 命令把所有依赖都安装好。
      • 使用pip freeze > requirements 可以快速将已安装的python包生成requirements文件。
    • README:项目说明文件
      • 软件定位，软件的基本功能
      • 运行代码的方式：安装环境，启动命令
      • 简要的使用功能
      • 代码目录结构，更详细点可以说明软件的基本原理
      • 常见问题说明
